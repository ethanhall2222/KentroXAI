KentroXAI Demo Kit

From one question to a governed, auditable AI evidence pack.

KentroXAI is not just a chat answer generator. It takes a question, retrieves evidence, generates an answer, evaluates answer truth, runs governance and red-team checks, and writes a full artifact package for review.

Chat app Databricks job Evaluation engine Red-team checks Explainability artifacts Scorecard and go/no-go
Executive Summary
1 Answer → 15+ Artifacts

The system's output is not only a response. It is a response plus metrics, traceability, governance gates, explainability artifacts, monitoring context, and an auditable manifest.

System Layers

Use this to explain the architecture before you touch the app.

1. Experience Layer

The user interface lives in apps/kentro-chat.

  • User asks the question
  • App returns answer and trust summary
  • Scorecard opens as the business-facing review surface

2. Orchestration Layer

The Databricks job and CLI coordinate execution.

  • rag_answer_trust_job.py
  • src/trusted_ai_toolkit/cli.py
  • Builds run folder and triggers artifact generation

3. Governance Layer

Evaluation, red-team, explainability, documentation, and release gates.

  • Answer-truth metrics
  • Controls and governance scores
  • Artifacts, monitoring, and incidents

Runtime Lifecycle

This is the end-to-end path you can walk through live.
Question and Answer

The user asks a question in the Kentro chat app. The Databricks-backed flow retrieves evidence chunks and generates an answer.

  • Question enters through apps/kentro-chat
  • Request reaches Databricks job backend
  • Answer and retrieved chunks become runtime inputs
Evaluation and Truth

The toolkit computes answer-truth metrics and support signals.

  • claim_support_rate
  • unsupported_claim_rate
  • contradiction_rate
  • evidence_sufficiency_score
  • context_relevance_embedding_coverage
  • advisory LLM judge metrics
Governance and Security

Release-facing controls and red-team checks run beside the answer metrics.

  • pillar scoring across security, reliability, transparency, governance
  • red-team findings and blocker counts
  • stage gates and required actions
Artifact Generation

The run ends by writing an evidence pack that can be reviewed or audited later.

  • scorecard HTML, JSON, and markdown
  • reasoning and lineage artifacts
  • manifest, cards, monitoring, and incident outputs

Key Demo Message

This is the one concept that keeps the entire story coherent.

Answer Trust

Is the answer itself supported by the available evidence?

Governance Status

Did the broader governed system pass release gates and blocker checks?

Important Distinction

A grounded answer can still fail governance. A governed system can still produce an untrusted answer.
